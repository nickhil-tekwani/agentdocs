# Repository instructions

- Read `SKILLS.md` and load `skills/agentdocs-repository/SKILL.md` for documentation work. AgentDocs is AI-client-agnostic: perform reasoning and writing in the user's chosen external AI tool, never by adding an embedded model provider.
- Keep `README.md` current. Update it whenever a sizeable change affects the repository's purpose, structure, setup, usage, behavior, or contributor workflow. If a change creates information that users or contributors would reasonably need from the README, include that update in the same change.
- Record meaningful technical or architectural decisions in `decisions_log.md` as they are made, including decisions reached during human–LLM sessions. Capture the decision, why it was made, and important alternatives or tradeoffs. Do not log routine edits or trivial implementation details.
- When revising an earlier decision, add a new entry that links to or names the superseded decision; preserve the historical entry.
